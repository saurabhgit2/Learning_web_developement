Still early. Still pushing commits anyway.

Most portfolios perform expertise. Mine is closer to a changelog — a record of actually learning to code, version by version. It started with the basics: setting up Git, pushing a first repo, breaking things locally before pushing them anywhere public.

From there: small Python programs to nail down logic and syntax, then a deliberate step into object-oriented design to move past "scripts that work" and toward "code that's structured." Every repository below is a real checkpoint, not a polished showcase — which is the point.
